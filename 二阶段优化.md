前端修改任务清单
一、修改点一：登录/注册功能完整集成（已修改）
当前问题： 登录/注册仅有界面占位，未对接真实逻辑，路由守卫被注释，后台无实际认证。

修改方式：

1.1 路由守卫恢复
修改 router/index.js，取消路由守卫注释，实现以下逻辑：

访问需要认证的页面（路由 meta 中 requiresAuth 为 true）时，检查 localStorage 中是否存在 admin_token

无 token 则跳转到 /login，并在 query 中携带 redirect 参数以便登录后跳回

已登录用户访问 /login 时自动跳转到首页

访问后台页面（/admin/*）时，额外检查 user_info 中的 role 字段，非 admin 或 auditor 角色跳转到首页

1.2 登录页对接
修改 Login.vue，实现：

用户名+密码表单，调用登录API（接口路径由后端提供）

登录成功后，将返回的 token 存入 localStorage 的 admin_token，用户信息存入 user_info

登录成功后跳转到 redirect 参数指定的页面或首页

登录失败显示错误提示

添加登录中 loading 状态防止重复提交

1.3 注册页对接
修改 Register.vue，实现：

用户名、密码、确认密码、角色选择（user/auditor/admin）表单

前端校验：用户名必填、密码不少于6位、确认密码与密码一致

调用注册API，成功后提示并跳转登录页

1.4 导航栏登录状态展示
修改 FrontLayout.vue，实现：

未登录时右上角显示"登录/注册"按钮，点击跳转 /login

已登录时右上角显示用户头像（取用户名首字母）+ 用户名下拉菜单

下拉菜单包含"个人中心"（占位）和"退出登录"

退出登录清除 localStorage 中的 admin_token、user_info 和 _captcha_verified，跳转首页

1.5 后台布局用户信息展示
修改 AdminLayout.vue，实现：

顶部栏显示当前登录用户名和角色标签（管理员/审核员）

"退出登录"清除所有认证信息并跳转首页

1.6 axios拦截器
修改 utils/request.js，实现：

请求拦截器：每次请求自动在 Header 中携带 Authorization: Bearer {token}

响应拦截器：收到 401 状态码时自动清除本地认证信息并跳转到登录页

二、修改点二：后台按钮级权限控制（已修改）
当前问题： 所有后台页面的新建、编辑、删除、批量删除、导入、导出按钮对所有访问者可见，无权限区分。

修改方式：

2.1 创建权限指令
新建 directives/permission.js，实现一个名为 permission 的自定义指令：

接收一个角色数组作为参数（如 ['admin'] 或 ['admin', 'auditor']）

在元素插入时检查当前登录用户的 role 是否在允许列表中

不在列表中则移除该 DOM 元素

2.2 应用到所有后台管理页
对以下文件逐一修改：HerbManage.vue、PresManage.vue、SymptomManage.vue、SyndromeManage.vue、RelationManage.vue、DocManage.vue、RecordManage.vue

"新建"按钮：添加 v-permission="['admin', 'auditor']"

"编辑"按钮（表格操作列）：添加 v-permission="['admin', 'auditor']"

"删除"按钮（表格操作列）：添加 v-permission="['admin']"

"批量删除"按钮：添加 v-permission="['admin']"

"批量导入"按钮：添加 v-permission="['admin', 'auditor']"

"导出"按钮：添加 v-permission="['admin', 'auditor']"

2.3 后台侧边栏菜单权限
修改 AdminLayout.vue 中的侧边栏，实现：

根据用户角色动态显示菜单项

admin 和 auditor 可见所有管理菜单

user 角色不可见后台菜单（或后台整体不可访问）


三、修改点三：文献管理增加溯源字段（已改动）
当前问题： 文献管理只有标题、类型、内容、来源、关联实体字段，缺少原文片段、页码章节、来源详情等溯源信息，RAG回答无法展示依据。

修改方式：

3.1 文献表单新增三个字段
修改 DocManage.vue 的新建/编辑弹窗表单：

增加"来源详情"文本输入框，字段名 source_detail，用于记录具体来源（如"《中国药典》2020版 第一卷"）

增加"原文片段"多行文本输入框，字段名 original_text，用于直接引用原文内容，RAG回答中展示使用

增加"页码/章节"文本输入框，字段名 chapter（如"P123"或"第五章"）

这三个字段放在"文献类型"之后、"内容"之前

3.2 文献表格新增两列
修改 DocManage.vue 的数据表格：

在"文献标题"列后增加"来源类型"列，用不同颜色的 el-tag 显示（药典=红色、教材=蓝色、古籍=橙色、期刊=绿色、科普=灰色）

增加"原文片段"列，截断显示前30字加省略号，鼠标悬停用 el-tooltip 显示全文

3.3 详情弹窗展示溯源信息
修改文献的"查看详情"弹窗，在内容上方展示来源类型、来源详情、页码/章节、原文片段等溯源信息

四、修改点四：ChatBubble增加RAG依据展示（已改动，等待后端返回数据）
当前问题： RAG回答中的引用资料来源（sources）字段已在API返回数据中，但前端未展示，用户看不到回答的依据。

修改方式：

4.1 ChatBubble组件增加依据面板
修改 ChatBubble.vue：

在消息内容下方新增一个折叠面板（el-collapse），仅当消息来源非用户且 sources 数组有内容时显示

折叠面板标题为"📚 查看依据"，默认收起

展开后遍历 sources 数组，每项展示：

来源类型标签（颜色同文献管理：药典=红/教材=蓝/古籍=橙/期刊=绿/科普=灰）

文献标题

来源详情（source_detail）

原文片段（original_text），用浅灰背景+左侧蓝色边框引用样式展示

关联实体列表（如果有），以 el-tag 小标签展示，点击可触发跳转到图谱搜索

4.2 ChatAI父组件传递sources
修改 ChatAI.vue，在接收到API回答时，将返回的 sources 字段透传给 ChatBubble 组件


五、修改点五：首页统计卡片调整（无需调整）
当前问题： 首页统计区只显示了药材、方剂、关系、问答四个统计项，可以更全面地展示平台数据规模。

修改方式：

修改 Index.vue 的统计区：

将当前4卡片布局改为6卡片，新增"症状"和"证候"两个统计卡片

保持每个卡片点击跳转到对应后台管理页的功能

数据来源仍然使用 entityApi 获取，API失败时使用mock数据兜底

六、修改点六：路由权限标记补全
当前问题： 部分需要登录才能访问的页面路由未标记 requiresAuth，存在绕过登录直接访问的风险。

修改方式：

修改 router/index.js，为以下路由添加 meta.requiresAuth: true：

所有后台管理页面（/admin/* 所有子路由）

历史记录页面（/history）

病例教学页面（/case-study）

注意：首页（/）、智能问答（/chat）、知识图谱（/graph）不需要登录即可访问

七、修改点七：人机验证与登录的关系梳理
当前问题： 进入后台目前只需要人机验证（数学题），与登录认证是两套独立机制，逻辑混乱。

修改方式：

保持人机验证作为进入后台的额外安全门槛，但必须在已登录的前提下才触发

即：点击"后台管理"时，先检查是否已登录，未登录则跳转登录页；已登录则弹出人机验证弹窗

验证通过后才跳转后台页面

验证状态仍用 sessionStorage 存储，同一浏览器会话有效

🔴 P0	修改点一：登录/注册功能完整集成	后台安全的基础，没有认证一切权限控制都无从谈起。当前"后台管理"仅靠数学题验证，形同虚设
🔴 P0	修改点六：路由权限标记补全	与P0-1配套，不标记则未登录用户仍可直接访问后台页面
🔴 P0	修改点七：人机验证与登录的关系梳理	与P0-1/P0-6配套，理顺验证逻辑，避免用户困惑
🟡 P1	修改点二：后台按钮级权限控制	登录有了之后，不同角色看到不同按钮是基本要求，否则审核员也能删除数据
🟡 P1	修改点三：文献管理增加溯源字段	溯源是RAG可解释性的核心需求，且涉及数据库字段变更，越早做越好
🟢 P2	修改点四：ChatBubble增加RAG依据展示	依赖修改点三（sources数据来源），可在后端接口ready后再做

简要说明
P0 三个修改点（登录+路由+人机验证）必须一次性全部完成，否则后台权限体系形同虚设，建议作为第一批次统一交付。做好后后台才真正安全。

P1 两个修改点（按钮权限+文献溯源） 是核心功能增强。按钮权限直接服务于P0建立的权限体系，文献溯源则涉及数据模型变更，建议作为第二批次尽快完成。

P2 两个修改点（RAG依据展示+首页卡片） 属于锦上添花，可在前后端联调阶段视时间情况决定是否纳入本次迭代。
