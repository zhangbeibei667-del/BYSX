# 中医药知识图谱关系约束表

严格约束的代码来源为 `server/schemas.py`。生产接口和严格导入模式不得自动接受未知关系组合。

| 关系 | 起点类型 | 终点类型 | 方向说明 | 证据要求 |
|---|---|---|---|---|
| 包含 | 方剂 | 药材 | 方剂由药材组成 | 必须 |
| 主治 | 方剂、药材 | 症状、证候、疾病 | 治疗知识关联 | 必须 |
| 提示 | 症状 | 证候 | 症状对辨证的提示 | 必须 |
| 对应 | 证候 | 方剂 | 教学范围内的证候—方剂关联 | 必须 |
| 常见证候 | 疾病 | 证候 | 某疾病常见辨证分型 | 必须 |
| 具有 | 药材、方剂 | 功效 | 功效关联 | 必须 |
| 具有 | 药材 | 性味 | 兼容现有性味数据 | 必须 |
| 性味 | 药材 | 性味 | 推荐的新数据写法 | 必须 |
| 归经 | 药材、方剂 | 归经 | 归经关联 | 必须 |
| 禁忌 | 药材、方剂 | 禁忌、证候、症状 | 禁用或慎用条件 | 必须 |
| 来源于 | 任意正式知识实体 | 文献 | 知识来源 | 必须 |
| 记载 | 文献 | 药材、方剂、症状、证候、功效、禁忌、疾病 | 反向文献关联 | 必须 |

## 严格校验

- 未知实体类型：严格导入时拒绝。
- 未知关系类型：拒绝。
- 不符合表中方向的关系：拒绝。
- 源实体或目标实体不存在：拒绝。
- 源实体与目标实体相同：拒绝。
- `reviewed`、`approved` 关系无证据：拒绝。
- 关系唯一键：`source_id + relation + target_id`。

## 疾病关系说明

原数据使用“疾病—主治—证候”，语义方向不准确，现已统一为：

```text
疾病 --常见证候--> 证候
```

“主治”只用于方剂或药材指向症状、证候、疾病。
