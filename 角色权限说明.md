# 角色权限说明

## 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| 普通用户 | `user` | 可查看和导出后台数据，无写入权限 |
| 管理员 | `admin` | 拥有全部后台管理权限（新建、编辑、删除、导入） |

## 后台页面权限矩阵

| 操作 | `admin` | `user` | 说明 |
|------|---------|--------|------|
| 查看列表 | ✅ | ✅ | 查看后台管理页面数据 |
| 查看详情 | ✅ | ✅ | 查看记录详情/文献内容 |
| 新建 | ✅ | ❌ | 创建新记录 / 上传文献 / 新增关系 |
| 编辑 | ✅ | ❌ | 修改已有记录 |
| 删除 | ✅ | ❌ | 删除单条记录 |
| 批量删除 | ✅ | ❌ | 批量删除多条记录 |
| 批量导入 | ✅ | ❌ | 从文件批量导入数据 |
| 导出 | ✅ | ✅ | 导出数据为 Excel 文件 |
| 收藏/取消 | ✅ | ✅ | 收藏问答记录 |

## 实现方式

### 权限指令 `v-permission`

```vue
<!-- 仅管理员可见 -->
<el-button v-permission="['admin']" @click="handleCreate">新建</el-button>

<!-- 管理员和普通用户均可见 -->
<el-button v-permission="['admin', 'user']" @click="handleExport">导出</el-button>

<!-- 所有角色可见的查看类按钮无需加指令 -->
<el-button @click="handleView">查看详情</el-button>
```

### 路由守卫

- `/admin/*` 路由需要登录（`meta.requiresAuth: true`），admin 和 user 均可访问
- `/history`、`/case-study` 需要登录
- `/`、`/chat`、`/graph` 无需登录
- 未登录用户访问受限页面会被重定向到 `/login?redirect=原路径`
- 按钮级别权限通过 `v-permission` 指令控制

### 关键文件

| 文件 | 作用 |
|------|------|
| `src/directives/permission.ts` | 权限指令定义（从 localStorage 读 user_info 判断角色） |
| `src/main.ts` | 全局注册 `v-permission` |
| `src/router/index.ts` | 路由守卫，检查登录状态和角色 |
| `src/store/index.ts` | 用户状态管理，角色类型定义 (`guest/user/admin`) |
| `角色权限说明.md` | 本文档，权限参考 |

### 后期扩展

如需新增角色或调整权限：
1. 修改 `src/store/index.ts` 中的 `UserRole` 类型
2. 更新本权限说明文档
3. 在各页面对应按钮上修改 `v-permission` 数组
