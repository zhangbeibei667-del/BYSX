前端完整功能模块详细阐释（完全匹配文档 5 大任务）
模块 1：前台教学交互模块（面向学生 / 中医学习者）
智能流式问答页面（核心 RAG 交互）
匹配任务 3、任务 4 图谱增强 RAG 与诊疗 Agent：
文字输入框，支持多行输入症状、药材、方剂提问；
SSE 流式接收后端 Agent 返回内容，打字机实时渲染 Markdown 答案；
回答下方自动展示图谱关联依据：相关药材 / 方剂 / 证候实体、关系路径、引用药典 / 文献片段，实现可解释问答；
自动追问功能：Agent 自动生成补充症状提问（如 “是否伴随畏寒？”），前端快捷追问按钮；
语音播放按钮：一键朗读方剂、证候解析内容；
对话历史侧边栏，支持新建对话、删除历史、收藏问答。
知识图谱可视化浏览页面
匹配任务 1、任务 2 图谱构建与图谱管理：
中央图谱画布：力导向布局，不同实体区分颜色节点；
顶部检索框：搜索药材 / 方剂，自动定位并高亮对应节点；
节点点击弹窗：展示实体全部属性（性味归经、功效、组成、禁忌）；
关系路径查询：选中两个实体，展示中间关联链路（如 “风寒感冒→证候→麻黄汤→包含麻黄”）；
缩放、拖拽、重置视图、导出图谱截图功能。
病例教学页面
文档要求病例教学输入功能：
表单录入患者症状、舌脉、基础信息；
提交后调用 Agent 自动匹配证候、推荐对应方剂；
侧边同步展示关联图谱节点，直观展示症状 - 证候 - 方剂关联；
保存病例模板，支持查看历史教学病例。
首页与历史记录页面
首页展示系统功能导航、热门药材 / 方剂推荐；历史页面统一管理所有问答、病例、收藏图谱节点。

模块 2：后台图谱数据管理模块（管理员维护图谱，匹配任务 2、任务 5）
全部采用统一 CRUD 页面模板，复用封装好的表格、搜索、弹窗组件，降低重复开发：
药材 / 方剂 / 症状 / 证候四大实体管理页
统一功能：
顶部搜索栏：按名称、功效、分类模糊检索；
表格展示全部数据，分页；
新增 / 编辑弹窗：表单录入实体全部属性（性味、归经、主治、用量、禁忌等）；
单条删除、批量删除；
Excel 批量导入、导出模板下载（图谱批量导入功能）。
图谱关系维护页面
专门管理实体之间关联关系（如方剂「包含」药材、证候「对应」症状、药材「具有」功效）：
下拉选择源实体、目标实体、关系类型；
新增 / 删除关系，表格展示全部三元组；
批量清除无效关系。
文献、问答记录管理页
文献：上传药典、中医教材文本，绑定对应图谱实体；
问答记录：查看所有用户提问与 Agent 回答，支持筛选、导出。
