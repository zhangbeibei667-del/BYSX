# 中医药知识图谱数据字典

## 1. 数据范围

当前正式图谱以项目根目录 `entities/` 与 `relations/` 为唯一数据源。

- 实体：2,005 个
- 关系：9,296 条
- 实体类型：10 类
- 关系类型：8 类
- 疾病实体：227 个，其中210个已建立疾病—证候关系

## 2. 实体公共字段

| 字段 | 类型 | 必填 | 约束 | 示例 |
|---|---|---:|---|---|
| `id` | string | 是 | 大写字母开头，只能包含大写字母、数字、`_`、`-`；同一图谱内唯一 | `H001` |
| `name` | string | 是 | 1–200字符；同类型内名称应唯一 | `人参` |
| `type` | string | 是 | 必须属于正式实体类型 | `药材` |
| `alias` | string | 否 | 多个别名使用 `/` 分隔 | `山参/棒槌` |
| `description` | string | 否 | 客观知识描述，不写诊疗承诺 | `五加科植物人参的干燥根` |
| `properties` | object | 否 | 类型专属属性及来源、版本、审核状态 | `{}` |

## 4. 关系字段

| 字段 | 类型 | 必填 | 约束 |
|---|---|---:|---|
| `source_id` | string | 是 | 必须引用已存在实体 |
| `source_name` | string | 是 | 应与 `source_id` 对应实体名称一致 |
| `relation` | string | 是 | 必须属于《关系约束表》定义的关系 |
| `target_id` | string | 是 | 必须引用已存在实体 |
| `target_name` | string | 是 | 应与 `target_id` 对应实体名称一致 |
| `evidence` | string | 是 | 至少包含资料名称和实体条目定位 |
| `evidence_level` | enum | 否 | `A_权威教材`、`B_药典标准`、`C_研究文献`、`D_经验总结`、`E_待验证` |
| `confidence` | number | 否 | 0–1 |
| `locator` | object | 否 | 可包含 `chapter`、`section`、`page` |
| `excerpt` | string | 否 | 可核验的短依据片段，不保存无授权长文本 |
| `version` | string | 否 | `主版本.次版本`，例如 `1.0` |
| `review_status` | enum | 否 | `draft`、`reviewed`、`approved` |

## 5. 审核规则

1. `draft` 可暂缺详细定位，但不得包含无法追溯的治疗结论。
2. `reviewed` 和 `approved` 必须填写 `evidence`。
3. 关系端点必须存在，禁止悬空关系和自环关系。
4. `approved` 数据修改后必须提升 `version` 并重新审核。
5. 项目所称“覆盖”仅指本数据字典列出的数据范围，不代表覆盖全部中医临床知识。
