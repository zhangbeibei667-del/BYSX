# 任务 2：图谱管理模块验收说明

## 完成状态

任务 2 按当前设计任务书所列功能完成度为 **100%**。默认使用完整的 Memory + JSON 持久化，无需安装或启动 MySQL、Neo4j、pymysql。MySQL、Neo4j、Hybrid 仍作为可选部署模式保留。

## 统一启动

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

统一应用同时提供：

- Agent、RAG 与教学接口：`/api/*`
- 图谱管理接口：`/api/kg/*`
- OpenAPI 文档：`/docs`
- 独立图谱管理器：`/kg-viewer`
- Vue 图谱浏览：`/graph`
- Vue 关系管理：`/admin/relations`

旧的独立图谱服务仍可通过 `python -m server.main` 启动，端口固定为 8001，不再与统一应用冲突。

## 首次登录与密钥

- 没有配置 `JWT_SECRET` 时，系统生成随机密钥并保存到 `data/.jwt_secret`。
- 没有配置 `ADMIN_PASSWORD` 时，系统生成随机管理员密码并保存到 `data/.admin_initial_password`。
- 以上两个文件以及 `data/store.json` 均已加入 `.gitignore`，不会提交到仓库。
- 正式部署建议通过环境变量配置 `JWT_SECRET`、`ADMIN_USERNAME`、`ADMIN_PASSWORD`。

## 可选存储切换

复制 `backend/.env.example` 中对应变量到实际运行环境，将 `STORE_BACKEND` 设置为 `mysql`、`neo4j` 或 `hybrid`。只有选择相应模式时才需要安装驱动并启动数据库。
