### 3.2 智能问答（[ChatAI.vue](src/views/Front/ChatAI/ChatAI.vue)）——路由 `/chat`

#### 页面功能
- **左侧对话历史栏**：展示用户的历史对话列表，支持切换、删除
- **右侧对话主区域**：
  - 空态展示 4 个示例问题按钮
  - 消息气泡（ChatBubble 组件）：Markdown 渲染 + 展开/收起 + 复制 + 收藏 + 追问
  - 流式打字动画效果
- **底部输入区**：textarea + 发送按钮 + 快捷模板弹窗 + 朗读按钮 + 清除按钮
- **快捷模板**：Popover 弹窗，分三个 Tab（症状咨询/方剂查询/药材辨析），每个 Tab 4 个模板

#### 交互流程

**发送消息流程：**
```
用户输入问题 → 按 Enter 或点击"发送"
  → addMessage('user', 问题)
  → addMessage('assistant', '', loading状态)
  → 三级兜底尝试获取回答：
      1. chatApi.askQuestion() → 成功则获取 AnswerResponse
      2. StreamSSE (EventSource) → 成功则获取 AnswerResponse  
      3. buildMockResponse(问题) → 本地关键词模拟
  → simulateStreamResponse() 逐字输出
  → 保存对话到 localStorage + chatApi.saveHistory()
```

**对话历史管理：**
- **新建对话**：保存当前对话 → 清空消息 → 重置 historyId
- **加载历史**：`chatApi.getHistoryDetail()` → 失败 → `localStorage` → 失败 → 提示
- **删除历史**：`chatApi.deleteHistory()` → 过滤列表 → 清理 localStorage
- **历史列表**：`chatApi.getHistory()` → 失败 → `localStorage`

**其他按钮：**
| 按钮 | 功能 |
|------|------|
| 朗读 | `window.speechSynthesis` 朗读最新 AI 回答（中文，语速 0.9） |
| 清除 | 保存当前对话 → 清空消息 → 重置 |
| 模板 | 打开 Popover 弹窗，选择后填入输入框并 focus |
| 示例问题 | 填入输入框并 focus |

**消息气泡（ChatBubble）按钮：**
| 按钮 | 功能 |
|------|------|
| 详情/收起 | 展开/收起结构化回答详情 |
| 复制 | 复制消息内容到剪贴板 |
| 收藏/已收藏 | `chatApi.favoriteQuestion()` → localStorage |
| 追问 | 将 `follow_up_questions` 填入输入框 |

### 3.3 知识图谱（[GraphBrowse.vue](src/views/Front/GraphBrowse/GraphBrowse.vue)）——路由 `/graph`

#### 页面功能
三栏布局：**左侧控制面板（20%）+ 中央画布（60%）+ 右侧详情面板（20%）**

#### 左侧面板功能

**① 实体搜索：**
- 输入框 + 搜索按钮
- 调用 `graphApi.searchEntities()` → 成功：合并节点到图谱 + 闪烁定位
- API 失败 → 本地模糊匹配 + 跳转到匹配节点

**② 实体筛选：**
- 4 个 checkbox（药材/方剂/症状/证候），默认全选
- 取消勾选 → 过滤图谱显示

**③ 图谱概览：**
- 实时显示当前可见节点数/关系数 + 颜色图例

**④ 关系路径查询（BFS）：**
- 起点下拉选择 + 终点下拉选择 + "查询路径"按钮
- 调用 `graphApi.findRelationPath()` → 成功：高亮路径
- API 失败 → 本地 BFS 算法查找
- "清除路径"按钮恢复默认视图

**⑤ 热门实体：**
- 按节点度数（连接数）排序 Top 6
- 点击跳转到该节点（居中 + 闪烁）

#### 中央画布功能

| 操作 | 行为 |
|------|------|
| 拖拽 | 平移画布 |
| 滚轮 | 缩放 |
| 点击节点 | 选中节点 → 右侧面板显示详情 → 高亮邻接关系 |
| 拖拽节点 | 移动节点位置 |
| 放大按钮 | `zoomLevel * 1.4`（上限 5） |
| 缩小按钮 | `zoomLevel / 1.4`（下限 0.1） |
| 重置按钮 | `dispatchAction('restore')` |
| 导出按钮 | `getDataURL('png')` → 下载 PNG 文件 |
| 定位标签 | 显示当前选中节点名称 |

#### 右侧详情面板功能
- 点击节点后从右侧滑入
- **属性信息**：展示节点的所有属性（如性味/归经/功效/组成等，中文映射标签）
- **关联关系**：列出该节点的所有边（源→关系→目标），点击可跳转到关联实体
- **收藏节点**：写入 localStorage（`graph_favorites`）
- **查看详情（后台）**：根据实体类型跳转：
  - 药材 → `/admin/herbs?editId=xxx`
  - 方剂 → `/admin/prescriptions?editId=xxx`
  - 症状 → `/admin/symptoms?editId=xxx`
  - 证候 → `/admin/syndromes?editId=xxx`
- **关闭按钮**：清除选中，收起面板

#### 从其他页面跳转到图谱
- 首页推荐卡片点击 → `/graph?search=实体名` → 自动搜索
- 病例教学证候/方剂/药材标签 → `/graph?search=关键词` → 自动搜索
- 历史记录图谱收藏 → `/graph?highlight=nodeId` → 定位节点

## 四、后台管理页面通用模式

所有 7 个后台页面共享相同的交互模式：

```
┌──────────────────────────────────────────────┐
│  页面标题          [新建] [批量导入] [导出]    │  ← 顶部操作栏
├──────────────────────────────────────────────┤
│  [搜索框] [分类筛选] [其他筛选] [搜索][重置] │  ← 搜索筛选栏
│  [批量删除(N)]                               │  ← 选中后出现
├──────────────────────────────────────────────┤
│  ☑ 序号 编号 名称  ...列...  操作(编辑/删除) │  ← 数据表格
│  ☐  1   H001 人参   ...      [编辑][删除]   │
│  ...                                         │
├──────────────────────────────────────────────┤
│              分页器                           │  ← 底部分页
└──────────────────────────────────────────────┘
```

**通用操作：**
| 操作 | 行为 |
|------|------|
| 新建 | 打开弹窗表单（dialog），填写后 `create()` |
| 编辑 | 打开弹窗表单（dialog），回填数据，修改后 `update()` |
| 删除 | 确认弹窗 → `delete(id)` |
| 批量删除 | 勾选 checkbox → "批量删除(N)" → 确认弹窗 → `batchDelete(ids)` |
| 搜索 | `list(params)` 重新加载表格 |
| 重置 | 清空搜索参数 → `list()` 加载全部 |
| 分页 | 切换页码 → `list({page, pageSize})` |
| 导入 | 打开 ImportDialog → 下载模板 → 上传 Excel → API `import()` |
| 导出 | API `export()` → Blob 下载 Excel |

**通用数据流：**
```
API 调用 (entityApi.xxx / relationApi / documentApi / recordApi)
  → 成功 → 更新 tableData + 提示
  → 失败 → 提示错误
```

## 六、数据持久化与兜底策略

由于后端未完全就绪，所有页面采用**API 优先 + localStorage 兜底 + mock 兜底**策略：

| 层级 | 说明 |
|------|------|
| L1: API | 优先调用后端接口（axios → `/api/*`） |
| L2: localStorage | API 失败时从浏览器本地存储读取 |
| L3: mock | 都没有时使用硬编码的模拟数据 |

| 数据 | localStorage Key | 使用者 |
|------|-----------------|--------|
| 对话历史列表 | `tcm_chat_history` | ChatAI, History |
| 对话消息 | `tcm_chat_messages_{id}` | ChatAI |
| 问答收藏 | `tcm_chat_favorites` | History |
| 图谱收藏 | `graph_favorites` | GraphBrowse, History |
| 病例数据 | `tcm_cases` | CaseStudy, History |
| 验证状态 | `_captcha_verified`（sessionStorage） | FrontLayout |
| 后台 Token | `admin_token` | axios 拦截器, AdminLayout |

## 七、关于身份认证

- **前端没有登录/注册功能**："登录/注册"按钮仅 `console.log('登录')`
- **路由守卫已注释**：`router.beforeEach` 中 `meta.requiresAuth` 检查被注释
- **admin_token**：仅在 `localStorage` 中作为占位存在，`AdminLayout` 的"退出登录"清除它
- **人机验证**：进入后台的唯一"门槛"，基于 sessionStorage 的数学题验证，非身份认证
