# 登录页后期修改说明

> 当前登录/注册页面已实现前端 UI 和模拟逻辑。待后端用户系统（含角色权限）就绪后，需要在本项目以下位置进行修改。

---

## 1. 路由守卫 — 启用权限拦截

**文件：** [src/router/index.ts](src/router/index.ts#L109-L122)

取消注释 `beforeEach` 并完善：

```ts
import { useUserStore } from '@/store'

router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  const token = localStorage.getItem('admin_token')

  // 需要登录的页面 → 未登录跳转登录页
  if (to.meta.requiresAuth && !token) {
    next({ path: '/login', query: { redirect: to.fullPath } })
    return
  }

  // 后台页面 → 仅 admin 可访问
  if (to.path.startsWith('/admin') && !userStore.isAdmin) {
    next('/')
    return
  }

  next()
})
```

同时在后台路由上启用 `meta: { requiresAuth: true }`（代码中已标注，取消注释即可）。

## 2. 导航栏 — 登录后显示用户名 / 退出按钮

**文件：** [src/layouts/FrontLayout.vue](src/layouts/FrontLayout.vue#L17)

```vue

<el-button v-if="!userStore.isAuthenticated" text class="login-btn" @click="login">
```

```ts

const handleLogout = () => {
  userStore.logout()

}
```

## 4. API 响应拦截器 — 401/403 自动跳转登录

**文件：** [src/api/index.ts](src/api/index.ts#L32-L38)

当后端返回 401（未认证）或 403（无权限）时，自动清除 token 并跳转登录页：

```ts
api.interceptors.response.use(
  (response) => response.data,
  (error) => {
    // 401 未认证 → 清除 token，跳转登录页
    if (error.response?.status === 401) {
      localStorage.removeItem('admin_token')
      // 避免在登录页自身触发时死循环
      if (!window.location.pathname.startsWith('/login')) {
        window.location.href = '/login'
      }
    }
    // 403 无权限 → 提示用户
    if (error.response?.status === 403) {
      console.error('无权限访问该资源')
      // 可用 ElMessage.warning('无权限访问') 提示（需在组件上下文中调用）
    }
    return Promise.reject(error)
  }
)
```


### 5.2 对接真实 API

替换模拟代码，调用 `authApi`：

```ts
import { authApi } from '@/api'

// 登录 —— 替换模拟代码
const res: any = await authApi.login({
  username: loginForm.username,
  password: loginForm.password
})
userStore.login(res.data.token, res.data.user)

// 注册 —— 替换模拟代码
const res: any = await authApi.register({
  username: registerForm.username,
  email: registerForm.email,
  password: registerForm.password
})
userStore.login(res.data.token, res.data.user)
```

## 6. Store — 完善 checkAuth 方法

**文件：** [src/store/index.ts](src/store/index.ts#L29-L35)

`checkAuth` 目前只检查了 token 是否存在，后续可改为调用后端接口验证 token 有效性：

```ts
import { authApi } from '@/api'

const checkAuth = async () => {
  const token = localStorage.getItem('admin_token')
  if (!token) return

  try {
    const res: any = await authApi.getCurrentUser()
    isAuthenticated.value = true
    userInfo.value = res.data.user
  } catch {
    // token 失效，清除
    localStorage.removeItem('admin_token')
    isAuthenticated.value = false
  }
}
```

并在应用启动时调用（可在 [src/main.ts](src/main.ts) 或 `App.vue` 的 `onMounted` 中）：

```ts
import { useUserStore } from '@/store'
const userStore = useUserStore()
userStore.checkAuth()
```
