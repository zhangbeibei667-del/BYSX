# 二阶段优化前端流程

> 中医药诊疗智能体平台 — 前端逻辑、页面功能与跳转关系分析文档

---

## 一、项目概述

项目前端基于 **Vue 3 + TypeScript + Element Plus + ECharts + Axios** 构建，分为**前台展示区**（FrontLayout）和**后台管理区**（AdminLayout）两大模块。前台面向普通用户，提供中医药智能问答、知识图谱浏览、病例教学等功能；后台面向管理员/用户，提供药材、方剂、症状、证候、图谱关系、文献、问答记录等数据的管理维护。

---

## 二、整体架构

### 2.1 路由结构

路由定义在 [src/router/index.ts](src/router/index.ts)，采用两层嵌套结构：

```
/                          → FrontLayout（前台布局）
├── /                      → FrontIndex（首页）
├── /chat                  → ChatAI（智能问答）
├── /graph                 → GraphBrowse（知识图谱浏览）
├── /case-study            → CaseStudy（病例教学）   [需登录]
├── /history               → History（历史记录）      [需登录]
├── /login                 → Login（登录/注册）

/admin                     → AdminLayout（后台布局）  [需登录 + 角色验证]
├── /admin/herbs           → HerbManage（药材管理）
├── /admin/prescriptions   → PresManage（方剂管理）
├── /admin/symptoms        → SymptomManage（症状管理）
├── /admin/syndromes       → SyndromeManage（证候管理）
├── /admin/relations       → RelationManage（图谱管理）
├── /admin/documents       → DocManage（文献管理）
├── /admin/records         → RecordManage（记录管理）

/:pathMatch(.*)*           → 重定向到 /
```

### 2.2 路由守卫（权限控制）

定义在 [src/router/index.ts:117-157](src/router/index.ts#L117-L157)：

| 条件 | 行为 |
|------|------|
| 目标路由 `meta.requiresAuth = true` 且无 token | 跳转 `/login?redirect=目标路径` |
| 目标路由以 `/admin` 开头 | 额外检查 `user_info.role` 是否为 `admin` 或 `user`，否则跳转 `/` |
| 已登录用户访问 `/login` | 自动跳转 `/` |

### 2.3 API 层

定义在 [src/api/index.ts](src/api/index.ts)，使用 Axios 实例，baseURL 为 `/api`，自动携带 Bearer token。

| 模块 | 关键接口 | 用途 |
|------|---------|------|
| `authApi` | login / register / logout / getCurrentUser / changePassword | 用户认证 |
| `chatApi` | askQuestion / askQuestionStream / getHistory / getHistoryDetail / saveHistory / deleteHistory / favoriteQuestion | 智能问答对话 |
| `graphApi` | getFullGraph / searchEntities / getEntityDetail / findRelationPath / getRelatedEntities | 知识图谱操作 |
| `literatureApi` | getDetail / getByEntity / search | 文献溯源 |
| `entityApi.herbs` | CRUD + import/export | 药材管理 |
| `entityApi.prescriptions` | CRUD + import/export | 方剂管理 |
| `entityApi.symptoms` | CRUD + import/export | 症状管理 |
| `entityApi.syndromes` | CRUD + import/export | 证候管理 |
| `relationApi` | CRUD | 图谱关系管理 |
| `documentApi` | CRUD + upload/export | 文献管理 |
| `recordApi` | CRUD + favorite/export/stats | 问答记录管理 |
| `caseApi` | create / analyze / list / detail | 病例管理 |
| `statsApi` | getPlatformStats / getPopularEntities / getDailyQuestions / getCategoryStats | 平台统计 |

### 3.3 智能问答 [ChatAI.vue](src/views/Front/ChatAI/ChatAI.vue)

**功能：** AI 对话式的智能问答页面，支持中医药相关问题的咨询。这是平台的核心交互页面。

**页面布局（三栏）：**
- **左侧栏：对话历史列表** — 显示历史对话，支持新建对话、加载历史对话、删除对话
- **中间：消息区域** — 聊天记录展示区
- **底部：输入区域** — 文本输入框 + 快捷模板

**功能操作：**

| 操作 | 触发方式 | 效果 |
|------|---------|------|
| 新对话 | 点击"新对话"按钮 | 清除当前对话，开始新会话 |
| 加载历史对话 | 点击历史列表项 | 加载该历史对话的消息记录 |
| 删除历史对话 | 点击历史项上的删除图标 | 删除该条对话记录（API + 本地） |
| 发送问题 | 输入框按 Enter 或点击"发送" | 发送问题 → 显示流式回答 |
| 使用示例问题 | 点击空状态下的示例按钮 | 将示例问题填入输入框 |
| 快捷模板 | 点击"模板"→ 选模板 | 填入模板问题（症状/方剂/药材三类） |
| 朗读回答 | 点击"朗读"按钮 | 调用 Web Speech API 朗读最新 AI 回答 |
| 清除对话 | 点击"清除"按钮 | 保存当前对话后清空 |
| 收藏回答 | 点击消息气泡的收藏按钮 | 调用 `chatApi.favoriteQuestion()` |
| 追问 | 点击消息气泡中的追问链接 | 将追问填入输入框 |
| 跳转图谱 | 点击回答中的实体名称 | → `/graph?search=实体名称` |

**回答内容包含（AnswerResponse 结构）：**
- `answer` — 回答正文（流式输出）
- `symptoms` / `syndromes` / `formulas` / `herbs` — 相关实体列表
- `evidence` — 循证溯源（文献引用）
- `sources` — 来源文献详情（含原文摘录、章节、关联实体）
- `follow_up_questions` — 追问建议
- `safety_notice` — 安全声明
- `graph` — 关联图谱数据

**API 调用链：**
1. 优先 `chatApi.askQuestion()`（普通 HTTP）
2. 兜底 `chatApi.askQuestionStream()`（SSE 流式）
3. 最终兜底本地模拟 `buildMockResponse()`

### 3.4 知识图谱浏览 [GraphBrowse.vue](src/views/Front/GraphBrowse/GraphBrowse.vue)

**功能：** 可视化展示中医药知识图谱，支持搜索、筛选、路径查询、节点详情查看。

**页面布局（三栏）：**
- **左侧面板（20%）：** 搜索框、实体类型筛选、图谱概览统计、关系路径查询、热门实体标签
- **中央画布（60%）：** ECharts 力导向图谱，支持拖拽、缩放、节点点击
- **右侧面板（20%）：** 选中节点的详情信息（属性列表、关联关系）

**功能操作：**

| 操作 | 入口 | 效果 |
|------|------|------|
| 搜索实体 | 左侧搜索框输入关键词 + 回车/点击搜索 | 搜索结果合并到图谱并高亮闪烁 |
| 类型筛选 | 左侧勾选/取消药材/方剂/症状/证候 | 图谱实时过滤显示 |
| 路径查询 | 左侧选起点实体 → 选终点实体 → 点击"查询路径" | 显示两实体间的关系路径，高亮路径 |
| 清除路径 | 路径查询后点击"清除路径" | 恢复完整图谱 |
| 节点点击 | 点击图谱中的节点 | 右侧面板显示详情 |
| 关联实体跳转 | 右侧面板点击关联关系项 | 跳转到该关联实体，居中+闪烁 |
| 热门实体跳转 | 左侧点击热门实体标签 | 跳转到该实体 |
| 缩放/重置/导出 | 右上角工具栏 | 缩放图谱、重置视图、导出PNG截图 |
| 收藏节点 | 右侧面板点击"收藏节点" | 保存到本地 `graph_favorites` |
| 查看后台详情 | 右侧面板点击"查看详情（后台）" | → `/admin/{对应类型页}?editId=节点ID` |
| 图谱节点点击（迷你事件） | ECharts click 事件 | 选中节点、加载详情 |

**图谱节点类型配色：**
| 类型 | 颜色 |
|------|------|
| 药材 | `#588264` (绿色) |
| 方剂 | `#8c6e4a` (棕色) |
| 症状 | `#c8a86e` (金色) |
| 证候 | `#b13e3e` (红色) |

**数据来源：**
- 完整图谱：`graphApi.getFullGraph()`，兜底模拟数据
- 搜索：`graphApi.searchEntities()`，兜底本地匹配
- 路径查询：`graphApi.findRelationPath()`，兜底本地 BFS 寻路
- 实体详情：`graphApi.getEntityDetail()`

### 3.5 病例教学 [CaseStudy.vue](src/views/Front/CaseStudy/CaseStudy.vue)

**功能：** 中医病例录入与智能辨证分析，辅助教学。支持患者信息填写、症状录入、AI 自动辨证分析（输出证候→方剂→药材推理链），附迷你关联图谱展示。

**页面布局（左右两栏）：**
- **左栏（50%）：** 病例录入表单
- **右栏（50%）：** 分析结果展示区

**表单字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | |
| 年龄 | 数字 | 0-150 |
| 性别 | 下拉 | 男/女 |
| 主诉与症状 | 文本域 | 最多2000字，核心分析输入 |
| 舌象 | 文本 | 如"舌淡苔白" |
| 脉象 | 文本 | 如"脉细弱" |

**功能操作：**

| 操作 | 入口 | 效果 |
|------|------|------|
| 智能分析 | 填写主诉后点击"🔍 智能分析" | 调用 `caseApi.create()` 创建病例 → `caseApi.analyze()` 分析 → 展示辨证结果 |
| 保存病例 | 点击"保存病例" | 保存到后端/本地 |
| 清空表单 | 点击"清空" | 确认后清空所有字段和分析结果 |
| 查看历史病例 | 点击"历史病例" | 展开底部横向滚动历史病例列表 |
| 加载历史病例 | 点击历史病例卡片 | 恢复该病例的表单和分析结果 |
| 跳转图谱 | 点击证候/方剂/药材标签 | → `/graph?search=标签名称` |

**智能分析输出：**
- **辨证论治图谱：** 迷你 ECharts 力导向图（症状→证候→方剂→药材）
- **证候诊断卡片：** 红色标签展示匹配证候 + 辨证分析文字
- **推荐方剂卡片：** 棕色标签展示推荐方剂 + 核心药材列表 + 典籍引证
- **教学笔记区：** 教师可记录教学要点

**本地模拟辨证覆盖的证型：**
心脾两虚、外感风寒、少阳证、血虚证、肝气郁结、肾阴虚、脾胃湿热、痰湿内阻、气虚证（基于关键词正则匹配）

### 3.7 登录/注册 [Login.vue](src/views/Front/Login/Login.vue)

## 七、关键数据流

### 7.1 登录流程

```
Login.vue → authApi.login() → 获取 { token, user }
→ userStore.login(token, user) → localStorage 存储
→ router.push(redirect 或 '/')
→ FrontLayout 读取 userStore.isAuthenticated → 显示用户信息
```

### 7.2 智能问答流程

```
ChatAI.vue → 用户输入问题
→ chatStore.addMessage('user', question)
→ chatApi.askQuestion(question, historyId)
   → 兜底: SSE 流式 / 本地模拟
→ simulateStreamResponse() → 逐字显示
→ chatStore.endStream() → 保存到 localStorage + 服务端
→ 显示 AnswerResponse（含溯源文献、追问建议、关联图谱）
```

### 7.3 图谱操作流程

```
GraphBrowse.vue → graphApi.getFullGraph() → ECharts渲染力导向图
→ 用户搜索 → graphApi.searchEntities() → 结果合并到图谱
→ 用户点击节点 → graphApi.getEntityDetail() → 右侧面板展示
→ 路径查询 → graphApi.findRelationPath(source, target) → 高亮路径
→ 后台跳转 → router.push('/admin/{类型}?editId=节点ID')
```

### 7.4 病例教学流程

```
CaseStudy.vue → 用户填写表单 → 点击"智能分析"
→ caseApi.create(caseData) → 获取 caseId
→ caseApi.analyze(caseId) → 获取辨证结果
→ AnswerResponse 渲染证候/方剂/药材 + 迷你图谱
→ 点击标签 → router.push('/graph?search=标签')
```

## 八、技术要点

1. **国风设计统一**：全站使用统一的国风配色（深绿 `#2a4030`、金色 `#c8a86e`、米白 `#f7f3eb`），保持视觉一致性。

2. **API 兜底策略**：大部分页面采用"先调 API，失败则使用本地模拟数据"的策略，确保前端可独立运行和演示。

3. **localStorage 持久化**：对话历史、收藏、病例等数据除了服务端保存，也同步存到 localStorage 作为离线缓存。

4. **人机验证**：从前台进入后台管理时，通过 `CaptchaVerify` 组件弹出人机验证弹窗（滑块/点选验证），通过后才跳转。

5. **角色权限指令**：后台管理页通过 `v-permission` 自定义指令控制按钮可见性，admin 可写、user 只读。

6. **ECharts 图谱**：使用力导向布局（force layout），支持拖拽、缩放、节点高亮、路径高亮、闪烁动画、导出截图。

7. **流式输出**：问答使用 `simulateStreamResponse()` 模拟打字机效果，提升对话体验。
