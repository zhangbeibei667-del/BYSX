# 前后端接口联调报告

更新时间：2026-07-15

## 一、0 条关系问题

后端知识图谱实际包含关系数据，问题出在 `/api/graph/full` 的取数和前端筛选顺序不一致：

- 旧接口先从全量实体中截取 1,200 个节点，其中包含大量功效、禁忌、文献等实体。
- 新版前端只展示药材、方剂、症状、证候四类实体。
- 前端过滤节点后，旧接口返回的边两端没有同时保留，因此页面显示 749 个节点、0 条关系。

现已改为后端先选取前端可见的四类实体，再提取这些实体之间的关系。浏览器复验结果为 1,200 个节点、1,635 条关系，浏览器控制台无错误。

## 二、接口契约修复

检查了新版前端 API 封装与 FastAPI 路由，补齐以下兼容接口：

| 方法 | 接口 | 用途 |
| --- | --- | --- |
| POST | `/api/chat/favorite/{id}` | 收藏问答记录 |
| GET | `/api/literature/{id}` | 文献详情 |
| GET | `/api/literature/entity/{entity_id}` | 实体关联文献 |
| GET | `/api/literature/search` | 文献搜索 |

同时确认登录、问答、图谱、病例、实体管理、关系管理、资料管理、记录管理等前端调用均有对应后端路由和响应字段适配。

## 三、真实 HTTP 冒烟测试

所有请求均通过 Vite 前端代理 `/api` 调用后端，共执行 53 项，全部通过：

- 系统与认证：健康状态、登录、当前用户。
- 首页统计：平台统计、分类统计、热门实体、每日统计。
- 图谱：全图、搜索、实体详情、邻居、路径。
- 后台实体：药材、方剂、症状、证候的新增、查询、修改、导入、导出和删除。
- 关系：新增、查询和删除。
- 资料与文献：新增、列表、详情、修改、统计、上传、搜索、实体关联和删除。
- 病例：新增、列表、详情、分析和删除。
- 问答与 Agent：普通问答、SSE 流式问答、历史详情、收藏、会话列表。

测试产生的临时实体、关系、资料、病例和会话均已清理。
